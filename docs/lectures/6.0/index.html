'origin', which is your copy on GitHub.

              origin  git@github.com:eaton-lab/hack-the-planet.git (fetch)
              origin  git@github.com:eaton-lab/hack-the-planet.git (push)

Advanced git usage

Adding a new upstream remote. This is used when you fork a repo (create your own origin remote) but you still want to pull in changes from the original (upstream remote). 


              # add a new remote called 'upstream'
              git remote add upstream git@github.com:hackers-test/hack-the-planet.git
            

              origin  git@github.com:eaton-lab/hack-the-planet.git (fetch)
              origin  git@github.com:eaton-lab/hack-the-planet.git (push)
              upstream  git@github.com:hackers-test/hack-the-planet.git (fetch)
              upstream  git@github.com:hackers-test/hack-the-planet.git (push)

Advanced git usage

You can now call git pull {remote} {branch} to pull in updates from either your remote or the original (upstream). If those changes conflict with your changes then you will be asked to resolve them (we'll learn more on this later). 


              # pull in changes from the upstream remote
              git pull upstream main

What happens when you call a function from an object?

              oldvar = "  apples oranges and celery     "
              newvar = oldvar.strip(" ")
              newvar
            
 
              "apples oranges and celery"

string functions

Calling a function from a string object uses the object as the context for its operation. It is very similar to a bash program where the syntax is: program -option target. Here the program is strip(), the option is " " and the target is oldvar. 


              # call strip function to remove all " " characters from ends
              oldvar = "  apples oranges and celery     "
              newvar = oldvar.strip(" ")
              newvar
            
 
              "apples oranges and celery"

string functions

Another example. The function split will split one string into multiple strings. Because multiple strings is a different type of object the result is returned as a list. 


              # call split on a string
              oldvar = "  apples oranges and celery     "
              newvar = oldvar.split(" ")
              newvar
            
 
              ['', '', 'apples', 'oranges', 'and', 'celery', '', '', '', '', '']

chaining functions

You can apply a function directly to the result of another function. Here the strip() call returns a string with the blank character removed, and split() uses the new string as its target. This is shorter than creating an intermediate variable. 


              # call strip AND split on a string
              oldvar = "  apples oranges and celery     "
              newvar = oldvar.strip(" ").split(" ")
              newvar
            
 
              ['apples', 'oranges', 'and', 'celery']

Python: iterable challenge

Count the number of differences between these two DNA strings: 


              dna1 = "ACAGAGTTGCCAGGAGATGACAGAAAGGTGTGGGTTACAACTCTCTCTAATTTAAGGGCCAATTAACATT"
              dna2 = "ACAGAGTCGCCAGGAGATGACAGAAAGGTCTGGGTTACAACTCTCTCTAAAATAAGGGCCAATTAACGTT"

              diffs = 0
              nbases = len(dna1)
              for idx in range(nbases):
                  if dna1[idx] != dna2[idx]:
                      diffs = diffs + 1

              5

Python: iterable challenge

Count the number of differences between these two DNA strings: 


              dna1 = "ACAGAGTTGCCAGGAGATGACAGAAAGGTGTGGGTTACAACTCTCTCTAATTTAAGGGCCAATTAACATT"
              dna2 = "ACAGAGTCGCCAGGAGATGACAGAAAGGTCTGGGTTACAACTCTCTCTAAAATAAGGGCCAATTAACGTT"

              diffs = 0
              for i, j in zip(dna1, dna2):
                  if i != j:
