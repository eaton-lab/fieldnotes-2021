What is programming?

In the strict sense, it is a logical sequence of arguments to a computer so that it can perform a desired task.

In this class, however, we will focus on a broader definition of a program: This includes (1) human-readable source code (text) of the program instructions; (2) an open-access online repository to share and receive feedback on the source code from a community of users; (3) a design and set of installation instructions to make the program broadly accessible; and (4) detailed documentation.

What are programming languages?

A dialect we can use to provide instructions to the computer. Programming languages change over time, diverge from each other, or adopt from each other, just like spoken languages. The popularity of different languages has varied through time and among different communities of users.

In this class you will be introduced to several languages, including bash, HTML, Markdown, SVG, JavaScript, and Python; with greatest emphasis on bash and Python.

Our goals in this class

(1) To learn to design, code, and publish a software tool that can accomplish a useful task in biology; and (2) To learn technical and social skills to contribute to the biological data science community.

How is this class geared towards biologists?

Through our interactions and discussions.
When biological information is encoded as data, it becomes no different from any other type of data -- it is just text. The skills that we will focus on here will generally apply to any type of data. But, through our discussions in this class, among a community of people developing software projects to solve biological problems, you will be exposed to numerous computational approaches for solving biological problems.

Do I have the right kind of computer for this class?

Short answer: yes.
Scientific computing is overwhelmingly done on operating systems based on the Unix architecture (e.g., Linux, MacOSX), sometimes termed *nix systems. As research computing continues to move into the cloud, these systems which are more scalable for multi-user and multi-tasking are likely to remain and even grow in popularity. Because of this, Windows has historically been very difficult to use in research computing.

Fortunately, this has recently changed. As Microsoft has gained interest in cloud computing they have embraced Linux -- which for decades they had previously attacked as a competitor -- and Linux can now be installed and run on Windows10 as a completely contained additional operating system (Windows Subsystem for Linux 2).

So, if you are on Windows you will be learning Linux.

No shame, though, Linux has many of its own drawbacks.

Unix programs from the shell/terminal

A common syntax to call shell programs

 
# the full syntax, but some arguments can be left empty
> [program-name] -[options...] [target]

# no options or targets: shows files in the current dir
> ls

# option -l means to format the output with one file per line
> ls -l

# target is a filepath (show that dir instead of current dir)
> ls /home/

# options and target can be provided together
> ls -la /home/deren

Filepaths

In a terminal you should always know: where am I? where are my files?
There are unix programs to tell you these answers. 


# print working directory (current location)
> pwd

# change directory (move yourself to another target location)
> cd /home/deren

# list the files in this directory
> ls 

# show the filepath to the program `ls`
> which ls

The Unix filesystem

It is essential to understand the hierarchical structure of the filesystem on your computer. Directories (folders) are nested within directories, starting from the root (/).
Your first assignment will focus on learning the file hierarchy, and many or most of the errors you will encounter while learning to code will involve errors in writing filepaths. Learning to traverse the filesystem efficiently, in a terminal, will superpower your efficiency in programming.
